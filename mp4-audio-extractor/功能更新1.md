## 1. 更新概述

### 1.1 更新目标

在现有 MP4 视频音频提取功能基础上，新增**音频格式自由转换**功能，支持用户在 FLAC、MP3、WAV 三种格式之间任意转换，原文件完整保留，新音频输出至同级目录。

### 1.2 功能范围

| 维度 | 说明 |
|------|------|
| **源格式** | WAV、FLAC、MP3 |
| **目标格式** | WAV、FLAC、MP3 |
| **转换方式** | ffmpeg 命令行，按目标格式选择最优编码器 |
| **原文件** | 完整保留，不做任何修改 |
| **输出位置** | 原音频同级目录，文件名格式为 `{stem}.{target_ext}` |
| **防呆设计** | 源格式 = 目标格式时自动禁用转换按钮 |

### 1.3 设计原则

- **独立模块**：转换功能作为独立卡片模块，与视频提取互不干扰
- **风格统一**：沿用暗色科技风 UI 设计体系
- **输入安全**：全链路校验（文件存在、格式白名单、同格式拦截）
- **用户体验**：三选一 radio 选择器 + 一键转换 + 状态反馈

## 2. 需求分析

### 2.1 用户故事

> 作为用户，我已经从 MP4 视频中提取了 WAV 音频，现在想把它转成 FLAC 以便归档压缩，或转成 MP3 以便分享。我希望操作简单直观，原文件不被覆盖。

### 2.2 功能边界

- ✅ 支持 WAV ⟷ FLAC ⟷ MP3 之间的任意双向/多向转换
- ✅ 原音频文件保持不变，仅新增转换产物
- ✅ 转换产物放置于源文件同级目录
- ✅ 同格式转换自动拦截（WAV→WAV 无意义）
- ❌ 不支持批量 / 文件夹级转换（单次单文件）
- ❌ 不支持任何其他音频格式（OGG、AAC、OPUS 等）

### 2.3 与现有功能的差异

| 纬度 | 视频提取 | 音频转换 |
|------|----------|----------|
| 输入类型 | `.mp4` 视频 | `.wav` / `.flac` / `.mp3` 音频 |
| 输出容器 | 固定 `.wav` | 用户自选格式 |
| 核心操作 | 丢弃视频轨 → 解码音频 | 解码 → 按目标编码器重新编码 |
| 用户交互 | 选择文件 → 查看信息 → 提取 | 选择文件 → 选择目标格式 → 转换 |
| 额外 UI | 音频信息面板 | 格式选择器（三选一 radio） |

## 3. 技术选型

### 3.1 ffmpeg 编码器映射

不同的目标格式需要 ffmpeg 使用不同的音频编码器。映射关系如下：

| 目标格式 | ffmpeg `-acodec` 参数 | 额外参数 | 说明 |
|----------|----------------------|----------|------|
| WAV | `pcm_s16le` | — | 16-bit PCM 无压缩，通用性最强 |
| FLAC | `flac` | — | 无损压缩，文件大小约为 WAV 的 50%-70% |
| MP3 | `libmp3lame` | `-q:a 2` | VBR 高质量模式，文件最小 |

### 3.2 为什么是这三个格式

- **WAV**：无压缩原始音频，任何播放器/编辑软件均可直接使用
- **FLAC**：无损压缩，比特完美还原，长期归档首选
- **MP3**：最广泛的有损格式，兼容性无出其右，体积最小

三者覆盖了"通用 → 归档 → 分享"三类核心场景。

### 3.3 与视频提取的编码器对比

| 场景 | 编码器 | 本质 |
|------|--------|------|
| 视频提取（WAV） | `pcm_s16le` | AAC 解码 → PCM 重编码 |
| 音频转换（WAV） | `pcm_s16le` | 任意源 → PCM 重编码 |
| 音频转换（FLAC） | `flac` | 任意源 → FLAC 无损压缩 |
| 音频转换（MP3） | `libmp3lame` | 任意源 → MP3 有损编码 |

## 5. 后端设计

### 5.1 新增命令总览

在原有 `lib.rs` 的 4 个命令（`pick_file`、`extract_audio`、`get_audio_info`、`open_folder`）基础上新增 2 个命令：

```
lib.rs
├── pick_file()          → 原有，MP4 文件对话框
├── extract_audio()      → 原有，视频→WAV
├── get_audio_info()     → 原有，ffprobe 元数据
├── open_folder()        → 原有，打开资源管理器
├── pick_audio_file()    ★ 新增，音频文件对话框
└── convert_audio()      ★ 新增，格式转换核心
```

### 5.2 M1: pick_audio_file() — 音频文件对话框

```rust
#[tauri::command]
async fn pick_audio_file(app: tauri::AppHandle) -> Result<Option<String>, String>
```

**与 `pick_file()` 的差异仅在于文件过滤器**：

| 命令 | 过滤器 |
|------|--------|
| `pick_file` | `["mp4"]` |
| `pick_audio_file` | `["wav", "flac", "mp3"]` |

其余逻辑完全一致——调用原生对话框、返回 `Option<String>`、用户取消时返回 `None`。

### 5.3 M2: convert_audio() — 格式转换核心

```rust
fn convert_audio(input_path: String, target_format: String)
    -> Result<String, String>
```

#### 5.3.1 执行流程

```
输入: input_path (文件路径), target_format (目标格式字符串)
  │
  ├─[1] 文件存在性校验 ── Path::exists()
  │     └─ 不存在 → Err("文件不存在")
  │
  ├─[2] 源格式白名单校验 ── extension ∈ {"wav","flac","mp3"}
  │     └─ 不匹配 → Err("不支持的源格式")
  │
  ├─[3] 目标格式白名单校验 ── target_format ∈ {"wav","flac","mp3"}
  │     └─ 不匹配 → Err("不支持的目标格式")
  │
  ├─[4] 同格式拦截 ── src_ext == target_format
  │     └─ 相同 → Err("源格式与目标格式相同，无需转换")
  │
  ├─[5] 构建输出路径 ── {parent}/{stem}.{target}
  │
  ├─[6] 根据目标格式选择编码器
  │     ├─ "wav"  → ("pcm_s16le", [])
  │     ├─ "flac" → ("flac", [])
  │     └─ "mp3"  → ("libmp3lame", ["-q:a", "2"])
  │
  ├─[7] 动态构造 ffmpeg 命令
  │     ffmpeg -i {input} -y -acodec {codec} [extra_args] {output}
  │
  ├─[8] 执行并检查 exit status
  │     ├─ success → Ok(output_path)
  │     └─ failure → 清理残留文件, Err(...)
```

#### 5.3.2 动态命令构造设计

与 `extract_audio()` 使用固定参数数组不同，`convert_audio()` 中的 MP3 编码需要额外参数 `-q:a 2`。因此采用 `Command` 的链式调用方式动态添加参数：

```
cmd.arg("-i")           → 固定
cmd.arg(&input_path)    → 固定
cmd.arg("-y")           → 固定
cmd.arg("-acodec")      → 固定
cmd.arg(acodec)         → 动态（由 target_format 决定）
cmd.arg("-q:a").arg("2")→ 仅 MP3 时添加
cmd.arg(&output_str)    → 固定
```

#### 5.3.3 编码器选型依据

| 目标格式 | 编码器 | 选型理由 |
|----------|--------|----------|
| WAV | `pcm_s16le` | 最通用的 PCM 编码，16-bit 深度平衡质量与体积 |
| FLAC | `flac` | ffmpeg 内置 FLAC 编码器，无损压缩 |
| MP3 | `libmp3lame` | LAME 编码器，业界 MP3 编码金标准；`-q:a 2` 为 VBR 高质量档位（约 190 kbps） |

#### 5.3.4 错误处理矩阵

| 场景 | 错误信息 | 是否清理残留 |
|------|----------|-------------|
| 文件不存在 | `文件不存在: {path}` | — |
| 源格式不支持 | `不支持的源格式: .{ext}，仅支持 wav/flac/mp3` | — |
| 目标格式不支持 | `不支持的目标格式: .{ext}，仅支持 wav/flac/mp3` | — |
| 同格式 | `源格式与目标格式相同，无需转换` | — |
| ffmpeg 未安装 | `无法运行 ffmpeg，请确认 ffmpeg 已安装` | — |
| ffmpeg 执行失败 | `音频转换失败，ffmpeg 返回错误` | ✅ remove_file |

#### 同格式禁用检测

```javascript
function updateConvertButtonState() {
  const targetFormat = document.querySelector("input[name='convertFormat']:checked")?.value;
  const srcExt = selectedConvertPath
    ? selectedConvertPath.split(".").pop()?.toLowerCase()
    : null;

  btnConvert.disabled = !(
    selectedConvertPath          // 已选择文件
    && targetFormat              // 已选择目标格式
    && srcExt !== targetFormat   // 源格式 ≠ 目标格式
  );
}
```

**触发时机**：
1. 选择文件后 → `setConvertFile()` 调用
2. 切换 radio → `change` 事件监听

#### 转换执行

```javascript
btnConvert.addEventListener("click", async () => {
  const targetFormat = document.querySelector("input[name='convertFormat']:checked")?.value;
  setConvertLoading(true);

  try {
    const outputPath = await invoke("convert_audio", {
      inputPath: selectedConvertPath,
      targetFormat: targetFormat
    });
    showConvertStatus("success", "转换成功！已保存至: " + outputPath);
    btnConvertOpenFolder.classList.remove("hidden");
  } catch (e) {
    showConvertStatus("error", "转换失败: " + e);
  } finally {
    setConvertLoading(false);
  }
});
```

**加载态**：转换中禁用所有交互按钮（选择、清除、格式切换），显示 spinner。

## 8. 整体协调性保障

### 8.1 模块间契约

| 模块 | 输入 | 输出 | 错误处理 |
|------|------|------|----------|
| `pick_audio_file` | — | `Option<String>` | `Err(String)` |
| `convert_audio` | `(String, String)` — 文件路径 + 目标格式 | `String` — 输出路径 | `Err(String)` |

与原有命令保持完全一致的契约模式：`Result<T, String>` + 前端 `try/catch`。

### 8.2 状态隔离

两个功能模块使用完全独立的状态变量和 DOM 引用：

| 视频提取 | 音频转换 |
|----------|----------|
| `selectedFilePath` | `selectedConvertPath` |
| `setLoading()` | `setConvertLoading()` |
| `showStatus()` | `showConvertStatus()` |
| `hideStatus()` | `hideConvertStatus()` |
| `resetFile()` | `resetConvertFile()` |

这种设计确保两个功能卡片可以独立操作，互不干扰。用户在视频提取卡片进行操作的任何时候，都可以切换到转换卡片执行独立操作。

### 8.3 CSS 复用策略

### 8.4 命名约定一致性

- **HTML id**：驼峰命名，转换区域统一 `convert` 前缀 → `convertFilePlaceholder`、`convertStatusArea`
- **CSS class**：短横线命名，新增类名紧贴原有命名风格 → `.format-option`、`.section-title`
- **JS 函数**：驼峰命名，转换区域函数统一 `Convert` 中缀 → `setConvertFile()`、`showConvertStatus()`

### 8.5 错误处理统一

所有错误通过 Tauri 的 `Err(String)` 返回，前端通过两套独立的 `try/catch` 处理：

```
视频提取错误 → showStatus("error", msg)     → 红色提示条
音频转换错误 → showConvertStatus("error", msg) → 红色提示条
```

两种错误展示使用完全相同的 CSS 样式（`.status.error`），视觉一致。
