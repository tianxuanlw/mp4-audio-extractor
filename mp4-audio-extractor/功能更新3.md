## 1. 更新概述

### 1.1 更新目标

将音频分析的文字时序功能从"用户提供文字→VAD 等比例近似分配"升级为"whisper.cpp 精准语音识别→词级时间戳标注"。无论用户是否填写参考文本，都自动进行 ASR 识别并产生词级时序。有参考文本时作为 `--prompt` 传入 whisper，引导识别精度大幅提升。

此前功能更新 2 中仅有近似方案：用户必须填写文字才能产出时序，且时序为等比例估算。本次更新从根本上解决精准识别问题。

### 1.2 功能范围

| 维度 | 说明 |
|------|------|
| **无文字输入** | whisper.cpp 自动语音识别 → 词级时序标注（纯 ASR 模式） |
| **有文字输入** | whisper.cpp `--prompt` 引导识别 → 精准时序对齐（引导模式） |
| **ASR 引擎** | whisper.cpp v1.7.4 官方预编译二进制（零 Rust crate 依赖） |
| **模型** | ggml-tiny.bin (~77MB) / ggml-small.bin (~466MB) |
| **识别粒度** | 词级时间戳（whisper --word-thold 0.01） |
| **语言** | 自动检测，默认中文（`-l zh`） |
| **whisper 不可用** | 信号分析正常运行，文字时序图表显示安装提示，不阻塞整体流程 |

### 1.3 与旧版功能对比

| 维度 | 功能更新2（旧） | 功能更新3（新） |
|------|---------------|---------------|
| 无文字时的文本 | 无任何文本 | whisper 自动识别 |
| 有文字时的时序 | VAD 等比例近似分配 | whisper --prompt 词级精确时序 |
| 时序精度 | 估算级（±100ms 量级） | whisper 级（±10ms 量级） |
| 额外依赖 | 无 | whisper.cpp 二进制（外部进程调用） |
| Rust crate | 0 额外 | 0 额外（不改动 Cargo.toml） |

### 1.4 设计原则

- **零 Rust crate 依赖**：whisper.cpp 作为外部进程调用，不引入任何新的 Rust 库
- **渐进增强**：whisper 不可用时信号分析部分完全不受影响
- **安全优先**：锁定版本、固定下载源、SHA256 校验提示
- **引导优先模式**：有参考文本时精度大幅提升，无参考文本时自动工作

## 2. 需求分析

### 2.1 用户故事

> 作为用户，我在"音频分析"模块中拖入一段中文语音录音并点击分析。无论我是否知道这段录音的文字内容，我都希望能看到它被自动识别为文字，并且每个词都有精准的时间位置标注在时间轴上。如果我恰好知道录音的文字内容，填入后识别的时序应该更加精准。

### 2.2 功能边界

- ✅ 无文字输入时 whisper 自动 ASR 识别
- ✅ 有文字时 whisper --prompt 引导识别，时序更精准
- ✅ 词级时间戳（非句级、非段级）
- ✅ whisper 不可用时降级为友好提示，不报错阻塞
- ✅ 一键安装脚本（版本锁定 + SHA256 校验）
- ❌ 不支持实时流式识别（仅离线文件）
- ❌ 不支持说话人分离（Diarization）
- ❌ 不引入任何 Rust crate（不修改 Cargo.toml）

## 3. 技术选型

### 3.1 ASR 引擎选型

| 方案 | 方式 | 依赖 | 精度 | 选型 |
|------|------|------|------|------|
| whisper.cpp 外部进程 | Rust `std::process::Command` 调用预编译二进制 | whisper-cli.exe (~4MB) + 模型文件 (~77MB) | 高（Whisper tiny） | ✅ 选用 |
| whisper-rs crate | Rust FFI 绑定 whisper.cpp C++ 代码 | whisper-rs crate + 系统需 cmake/gcc 编译 | 高 | ❌ 引入了 Rust crate |
| 纯 Rust 手写 Transformer | 自行实现 Whisper 推理 | 模型文件，但引擎需手写 ~3000 行 | 取决于实现 | ❌ 工期 2-3 周 |
| VAD + 等比例分配 | 能量法 VAD → 按字数分配时序 | 无 | 低（估算级） | ❌ 精度不足 |

**选型理由**：`std::process::Command` 调用外部二进制是 Rust 标准库原生能力，不增加任何 crate 依赖，同时享受 whisper.cpp 社区的持续优化红利。

### 3.2 whisper.cpp 核心参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `-m` | 模型文件路径 | tiny/small 模型 |
| `-f` | 16kHz 单声道 WAV 文件 | 统一预处理格式 |
| `-l zh` | 中文 | whisper 自动检测实际语言 |
| `--output-json` | JSON 格式输出 | 结构化解析 |
| `--word-thold 0.01` | 词级时间戳阈值 | 越低越精细 |
| `--prompt` | 参考文本 | 引导识别提升精度 |

### 3.3 whisper JSON 输出结构

```json
{
  "transcription": [
    {
      "tokens": [
        {
          "text": "你",
          "offsets": { "from": 120, "to": 180 }
        },
        {
          "text": "好",
          "offsets": { "from": 180, "to": 250 }
        }
      ]
    }
  ]
}
```

Rust 解析此 JSON → 提取 `text` + `from`/`to`（毫秒）→ 转为秒 → `[{text, start, end}]` 数组。

### 3.4 音频预处理链路

```
原始音频 (wav/flac/mp3, 任意采样率/声道)
  └─ ffmpeg 预处理
     -ar 16000    → 重采样至 16kHz (whisper 要求)
     -ac 1        → 强制单声道
     输出临时 WAV → whisper-cli.exe 消费
  └─ 识别完成后删除临时文件
```

临时文件命名：`{原文件名}_whisper_temp.wav`，存放于原音频同级目录，识别完成后立即清理。

## 5. 后端设计

### 5.1 新增函数与命令总览

```
lib.rs 新增
├── find_whisper_exe()      — 内部函数，搜索 whisper-cli.exe 路径
├── find_whisper_model()    — 内部函数，搜索 ggml-*.bin 模型文件
├── transcribe_audio()      ★ Tauri 命令，whisper ASR + 词级时间戳
└── analyze_audio()         ★ 改造：集成 ASR 结果，移除旧 VAD+等比例方案
```

### 5.2 M2: find_whisper_exe() — 二进制路径搜索

```rust
fn find_whisper_exe() -> Result<String, String>
```

**搜索策略**：按优先级依次查找
1. `{exe目录}/tools/whisper-cli.exe`（生产部署）
2. `{exe目录}/../../../tools/whisper-cli.exe`（开发模式：target/debug → 项目根目录）

**设计理由**：Tauri 开发模式下 exe 位于 `src-tauri/target/debug/`，需要上溯三级目录到项目根。

### 5.3 M3: find_whisper_model() — 模型文件搜索

```rust
fn find_whisper_model() -> Result<String, String>
```

**搜索策略**：优先 small 模型（精度更高），回退 tiny 模型。

### 5.4 M4: transcribe_audio() — whisper ASR 命令

```rust
#[tauri::command]
fn transcribe_audio(input_path: String, reference_text: Option<String>)
    -> Result<String, String>
```

#### 5.4.1 执行流程

```
输入: input_path (任意音频), reference_text (可选参考文本)
  │
  ├─[1] 查找 whisper 二进制 + 模型文件
  │     find_whisper_exe()? + find_whisper_model()?
  │
  ├─[2] 音频预处理（ffmpeg → 16kHz 单声道 WAV 临时文件）
  │     ffmpeg -i {input} -ar 16000 -ac 1 -y {temp_wav}
  │     └─ 失败 → 清理临时文件 → Err
  │
  ├─[3] 构造 whisper 命令行
  │     whisper-cli -m {model} -f {wav} -l zh --output-json --word-thold 0.01
  │     [if reference_text: --prompt {text}]
  │
  ├─[4] 执行 → stdout JSON 字符串
  │     └─ 失败 → Err(stderr)
  │
  ├─[5] 清理临时 WAV 文件
  │
  ├─[6] 解析 JSON → 提取词级 [{text, start, end}] 数组
  │     from/to 毫秒 → 转为秒（3 位小数精度）
  │
  └─[7] 序列化为 JSON 字符串返回
```

#### 5.4.2 引导模式价值

有参考文本时 whisper 使用 `--prompt` 参数：

| 场景 | whisper 行为 | 时序精度 |
|------|-------------|----------|
| 无参考文本 | 自由识别，可能识别偏差（换词、丢词） | 中等 |
| 有参考文本 | 以 prompt 文本为引导，强制匹配已知内容 | **高（引导后 whisper 会优先匹配已知词序列）** |

### 5.5 M5: analyze_audio() 改造

**改造前（功能更新2）**：
```
if transcript.is_some():
    VAD 切分语音段 → 等比例分配文字 → text_alignment
else:
    text_alignment = []  // 无文字不产出时序
```

**改造后（功能更新3）**：
```
始终调用 transcribe_audio(input_path, transcript)
  ├─ 成功 → text_alignment = [{text, start, end}, ...]
  └─ 失败 → text_alignment = {"error": "..."}  // 降级容错
```

**关键变更**：
- `text_alignment` 字段从 `[]`（空数组）变为始终有值（ASR 结果或 error 对象）
- whisper 不可用时不报错，前端展示安装提示
- 旧的 VAD + `align_text_to_segments` 函数保留（代码未删除，但不再被 `analyze_audio` 调用）

### 6.2 M7: JS renderAllCharts() 逻辑更新

**改造前**：
```javascript
if (data.text_alignment && data.text_alignment.length > 0) {
    renderTextAlignment(data);  // 仅数组有效
}
```

**改造后**：
```javascript
const ta = data.text_alignment;
if (ta && Array.isArray(ta) && ta.length > 0) {
    renderTextAlignment(data);  // ASR 成功，渲染时序图表
} else if (ta && ta.error) {
    // whisper 未安装，Canvas 居中显示安装提示
    ctx.fillText("⚠ whisper 未安装，无法进行语音识别");
    ctx.fillText("请运行 tools\\setup-whisper.ps1 下载安装");
} else {
    chartTextSection.classList.add("hidden");
}
```

### 6.3 M8: whisper 不可用时的 UI

当 `text_alignment = {"error": "未找到 whisper-cli.exe..."}` 时，文字时序图表的 Canvas 绘制一条友好的安装提示，而非报错或留空。

## 8. 整体协调性保障

### 8.1 模块契约

| 模块 | 输入 | 输出 | 降级行为 |
|------|------|------|----------|
| `find_whisper_exe` | — | `String` (路径) | `Err` 告知安装方法 |
| `find_whisper_model` | — | `String` (路径) | `Err` 告知下载方法 |
| `transcribe_audio` | `(String, Option<String>)` | `String` (JSON) | `Err` 含 stderr 信息 |
| `analyze_audio` | `(String, Option<String>)` | `String` (JSON) | text_alignment = {error} 对象 |

### 8.2 容错设计

```
transcribe_audio 失败
  ├─ whisper 进程不存在 → analyze_audio 不受影响，text_alignment = {"error":"未找到 whisper..."}
  ├─ ffmpeg 预处理失败 → 返回 Err
  └─ whisper 识别失败 → 返回 Err（带 stderr）

analyze_audio 整体
  ├─ 信号分析失败 → 整体返回 Err
  └─ ASR 失败 → 信号分析正常返回，text_alignment 降级为 error 对象
```

**编排原则**：ASR 是可选的增强功能，信号分析是核心功能。ASR 失败不应阻塞信号分析。

### 8.3 与旧代码兼容

| 旧函数 | 状态 | 说明 |
|--------|------|------|
| `energy_based_vad()` | 保留，不再调用 | VAD 仍可用于其他场景 |
| `align_text_to_segments()` | 保留，不再调用 | 旧版时序函数 |
| `extract_pcm()` | 保留且复用 | `transcribe_audio` 不直接使用（whisper 需要原始 WAV），但 `analyze_audio` 仍用 |

### 8.4 CSS 复用

无新增 CSS。HTML 文案和 JS 逻辑更新，不改变样式。

## 9. 供应链安全设计

### 9.1 零 Rust crate 原则

本次更新**不修改** `src-tauri/Cargo.toml`，不引入任何新的 Rust crate 依赖。

whisper.cpp 作为外部进程调用，通过 Rust 标准库 `std::process::Command` 实现。这种设计的优势：

| 风险 | 缓解措施 |
|------|----------|
| 恶意 crate 供应链攻击 | ✅ 无新 crate，完全规避 |
| whisper.cpp 二进制被篡改 | ✅ 版本锁定 v1.7.4 + SHA256 校验提示 |
| 模型文件被投毒 | ✅ HuggingFace 官方仓库下载 |
| 运行时注入 | ✅ 沙箱隔离（whisper 是一个独立进程） |

### 9.2 setup-whisper.ps1 安全措施

| 措施 | 实现 |
|------|------|
| 版本锁定 | 固定下载 **v1.7.4** 发布版，不追最新 |
| 来源锁定 | GitHub Releases + HuggingFace（ggml 模型） |
| 完整性校验 | 脚本末尾提示 certutil SHA256 比对命令 |
| 最小化依赖 | 无 npm 安装、无 pip 安装、无 cargo 安装 |
| 信任根 | GitHub.com (Microsoft) + HuggingFace |
