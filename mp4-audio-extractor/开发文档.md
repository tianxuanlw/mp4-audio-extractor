## 1. 项目概述

### 1.1 需求描述

开发一个小工具，实现对 MP4 视频文件中的音频流进行无损提取（即不重新编码，直接复制音频流），并将提取出的音频文件放置在与源视频相同的目录下。

### 1.2 产出物

| 类型 | 说明 |
|------|------|
| **CLI 工具** | 纯 Rust 命令行工具，零外部依赖，支持批量和拖拽 |
| **GUI 应用** | 基于 Tauri v2 的桌面应用，含文件选择、音频信息预览、一键提取 |
| **拖拽脚本** | `提取音频.bat`，将 MP4 文件拖拽到脚本上即可提取 |

### 1.3 设计原则

- **最小依赖**：CLI 版本零外部 crate，仅依赖系统 ffmpeg
- **用户友好**：GUI 提供文件选择对话框、音频信息预览、操作状态反馈
- **健壮性**：输入校验、错误处理、边界情况（文件不存在、格式不支持）
- **安全性**：不修改源文件、不收集任何数据、输出路径清晰可控

### 2.1 核心技术

| 音频处理 | ffmpeg / ffprobe | 8.0.1 | 业界标准，`-acodec copy` 实现真正无损 |
| 包管理 | npm | 11.9.0 | Tauri CLI 通过 npm 安装 |
| 构建工具 | Cargo | 1.95.0 | Rust 原生构建系统 |

### 2.2 Crate 依赖

| Crate | 版本 | 用途 |
|-------|------|------|
| `tauri` | 2 | 桌面应用框架核心 |
| `tauri-build` | 2 | Tauri 构建脚本 |
| `tauri-plugin-dialog` | 2 | 原生文件选择对话框 |

| `serde_json` | 1 | JSON 序列化 |

> **注**：CLI 版本使用 `std` 标准库，无外部依赖。

### 2.3 为什么不用纯 M4A 直接复制而是 WAV？

最初方案是 `-acodec copy` 直接复制 AAC 流到 M4A 容器，完全无损。后改为 `-acodec pcm_s16le` 输出 WAV 格式：

| 方面 | M4A copy | WAV pcm_s16le |
|------|----------|----------------|
| 容器 | `.m4a` | `.wav` |
| 编码 | 原样复制 AAC | 解码为 PCM 16-bit |
| 文件大小 | 小（原始音频大小） | 大（无压缩） |
| 质量 | 完全无损（bit-exact） | 从 AAC 解码，不损失已有质量 |
| 兼容性 | 一般 | 极好，任何播放器可打开 |
| 适用场景 | 归档/二次处理 | 通用播放/进一步编辑 |

## 4. 后端设计

### 4.1 架构总览

Rust 后端采用 **Tauri Command 模式**，每个功能点封装为独立的 `#[tauri::command]` 函数，通过 `generate_handler![]` 宏注册到 Tauri 运行时。

```
lib.rs
├── pick_file()        → 异步，调用原生文件对话框
├── extract_audio()    → 同步，调用 ffmpeg 命令行
├── get_audio_info()   → 同步，调用 ffprobe 获取元数据
├── open_folder()      → 同步，调用 explorer 打开目录
└── run()              → 组装 Tauri Builder，注册插件和命令
```

### 4.2 M1: extract_audio() — ffmpeg 音频提取

```rust
fn extract_audio(input_path: String) -> Result<String, String>
```

**职责**：接收 MP4 文件路径，调用 ffmpeg 提取音频流并输出 WAV 文件。

**执行流程**：
1. 校验文件存在性 → `Path::new(&input_path).exists()`
2. 校验扩展名为 `.mp4` → 不区分大小写
3. 构建输出路径：`{parent_dir}/{stem}_audio.wav`
4. 构造 ffmpeg 命令：`ffmpeg -i {input} -vn -acodec pcm_s16le -y {output}`
5. 执行并检查 `exit status`
6. 成功返回输出路径，失败清除残留文件

**设计决策**：
- **`-vn`**：丢弃视频流，仅保留音频
- **`-y`**：覆盖已存在的输出文件，避免交互式确认
- **失败自动清理**：`std::fs::remove_file()` 防止残留 0 字节文件

### 4.3 M2: get_audio_info() — ffprobe 元数据

```rust
fn get_audio_info(input_path: String) -> Result<String, String>
```

**职责**：获取 MP4 文件中第一条音频流的元数据信息。

**ffprobe 参数解析**：
```
-v error                           # 仅输出错误
-select_streams a:0                # 选择第一条音频流
-show_entries stream=codec_name,   # 编码名称
    codec_long_name,sample_rate,   # 采样率
    channels,bit_rate              # 声道数、比特率
-of default=noprint_wrappers=1     # 简洁输出格式
```

**输出示例**：
```
codec_name=aac
codec_long_name=AAC (Advanced Audio Coding)
sample_rate=44100
channels=2
bit_rate=128000
```

前端 JS 负责将这些键值对解析为中文标签展示。

### 4.4 M3: pick_file() — 原生文件对话框

```rust
async fn pick_file(app: tauri::AppHandle) -> Result<Option<String>, String>
```

**设计要点**：
- 使用 `tauri_plugin_dialog::DialogExt` 提供的原生对话框
- 仅过滤 `.mp4` 文件
- `blocking_pick_file()` 同步等待用户选择
- 返回 `Option<String>`，用户取消时返回 `None`

### 4.5 M4: open_folder() — 打开资源管理器

```rust
fn open_folder(path: String) -> Result<(), String>
```

**设计要点**：
- 智能判断：文件路径则打开其父目录，目录路径则直接打开
- Windows 使用 `explorer` 命令

## 5. 前端设计

### 5.1 前端技术方案

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 框架 | 原生 HTML/CSS/JS | 工具型应用，无路由、无复杂状态管理 |
| 模块化 | ES Module (`type="module"`) | 作用域隔离，现代浏览器原生支持 |
| API 调用 | `window.__TAURI__.core.invoke()` | Tauri 原生桥接 |
| 对话框 | Tauri Plugin Dialog (Rust 侧) | 原生体验，避免浏览器 `input[file]` 限制 |

### 5.2 M5: 文件选择区域

**状态机设计**：

```
┌──────────────┐    选择文件     ┌──────────────┐
│ file-placeholder │ ──────────► │ file-selected  │
│   (默认显示)     │             │  (显示文件名)   │
└──────────────┘    清除选择     └──────────────┘
       ◄────────────────────────

触发条件：
- "选择文件"按钮 → 调用 Rust pick_file()
- "✕"清除按钮 → 重置所有状态
```

**CSS 关键：**
- 占位区：虚线边框 `border: 2px dashed rgba(255,255,255,0.2)`，传达"可放入"
- 选中条：淡紫色背景 `rgba(100,100,255,0.1)`，与主按钮色调呼应

### 5.3 M6: 音频信息面板

**布局**：CSS Grid 双栏网格 `grid-template-columns: 1fr 1fr`

**字段映射**（JS 层）：
| ffprobe key | 中文 Label | 格式化 |
|-------------|-----------|--------|
| codec_name | 编码格式 | 原文 |
| codec_long_name | 编码器 | 原文 |
| sample_rate | 采样率 | ÷1000 → "XX.X kHz" |
| channels | 声道数 | 原文 |
| bit_rate | 比特率 | ÷1000 → "XX kbps" |

**可见性**：默认 `hidden`，文件选择成功且获取到信息后显示。

### 5.4 M7: 主操作按钮

**三态管理**：

| 状态 | disabled | 文字 | 额外 |
|------|----------|------|------|
| 待选择 | `true` | "提取音频" | 半透明 |
| 可提取 | `false` | "提取音频" | hover 上浮+发光 |
| 提取中 | `true` | "提取中..." | spinner 动画 |

```javascript
function setLoading(loading) {
  btnExtract.disabled = loading;
  btnSelect.disabled = loading;   // 提取中禁用选择
  btnClear.disabled = loading;    // 提取中禁用清除
  // 切换文字和 spinner
}
```

**防重复提交**：提取过程中 `btnSelect`、`btnClear` 全部禁用。

### 5.5 M8: 状态提示区域

**两种变体**：
- `.status.success` — 绿色系 `rgba(0,200,100,0.1)`
- `.status.error` — 红色系 `rgba(255,80,80,0.1)`

成功时额外显示"打开所在文件夹"按钮。

## 6. 前后端通信

### 6.1 Tauri Command 注册

```rust
// lib.rs
.invoke_handler(tauri::generate_handler![
    pick_file,      // 异步
    extract_audio,  // 同步
    get_audio_info, // 同步
    open_folder     // 同步
])
```

### 6.2 前端调用

```javascript
// main.js
const { invoke } = window.__TAURI__.core;

// 文件选择
const path = await invoke("pick_file");

// 音频提取
const outputPath = await invoke("extract_audio", { inputPath: selectedFilePath });

// 获取信息
const info = await invoke("get_audio_info", { inputPath: path });

// 打开文件夹
await invoke("open_folder", { path });
```

### 6.3 数据流

```
用户点击"选择文件"
  → main.js: invoke("pick_file")
    → lib.rs: pick_file(app) → 原生对话框
      → 返回文件路径 (String | null)
        → main.js: 更新 file-selected 显示
          → main.js: invoke("get_audio_info")
            → lib.rs: ffprobe 解析
              → main.js: 渲染 info-grid

用户点击"提取音频"
  → main.js: invoke("extract_audio")
    → lib.rs: ffmpeg 处理
      → 返回输出路径
        → main.js: 显示成功 + "打开文件夹"按钮
```

## 7. 整体协调性保障

### 7.1 模块间契约

每个模块定义了明确的输入/输出契约：

| 模块 | 输入 | 输出 | 错误处理 |
|------|------|------|----------|
| extract_audio | `String` (文件路径) | `String` (输出路径) | `Err(String)` |
| get_audio_info | `String` (文件路径) | `String` (键值对文本) | `Err(String)` |
| pick_file | — | `Option<String>` | `Err(String)` |
| open_folder | `String` | `()` | `Err(String)` |

所有 Rust 函数统一使用 `Result<T, String>` 返回，前端通过 try/catch 统一处理错误。

### 7.2 CSS/HTML/JS 命名约定

- **HTML id**：驼峰命名，语义化 — `btnExtract`、`filePlaceholder`、`audioInfo`
- **CSS class**：短横线命名，表示状态 — `.file-placeholder`、`.btn-primary`、`.status.success`
- **hidden 控制**：统一通过 `.hidden { display: none !important; }` 切换可见性

### 7.3 状态一致性

前端通过一个中央变量 `selectedFilePath` 串联所有 UI 状态：

```
selectedFilePath
├── null → filePlaceholder 显示, btnExtract disabled
└── "path" → fileSelected 显示, audioInfo 加载, btnExtract enabled
```

`setLoading(true)` 时，所有交互按钮统一禁用，防止并发操作冲突。

### 7.4 错误处理策略

```
┌──────────────┐
│   Rust 后端   │ → Result<T, String>
└──────┬───────┘
       │ Ok(T)
       ▼          ┌──────────────┐
   正常流程       │  前端 try/catch │
                  └──────┬───────┘
       │ Err(e)          │
       ▼                 ▼
   showStatus("error", e)  ← 统一错误展示
```
